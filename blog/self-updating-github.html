My GitHub profile rebuilds itself every morning.

Navdeep Kumar · Berlin · 2026

If you open my GitHub profile, an ASCII portrait of me types itself onto the screen, a terminal-style info card prints next to it, and a contribution heatmap animates in above both. None of it uses JavaScript, because GitHub strips scripts from profile READMEs.

The trick: GitHub does render SVG images, and it does play the animations inside them. So all the motion lives inside three SVG files, and the README just places them. I first saw this approach in a post by Avi Vashishta and adapted it for my own profile.

The three files

  • The portrait. A photo runs through background removal and contrast boosting, then gets downsampled to a grid where each pixel picks a character from a density ramp. Each row wipes in left to right with a small cursor riding the edge. One color only. Rainbow ASCII art always looks like noise.
  • The info card. Hand-written SVG styled like the output of neofetch: my role, my projects, my stack, fading in line by line.
  • The heatmap. The interesting one, because it has to stay fresh.

A live contribution graph with no token

GitHub serves everyone's contribution calendar as public HTML at github.com/users/<name>/contributions. No API key, no GraphQL, no rate-limit drama. A small Python script fetches that page, parses the day cells, and writes a JSON file. A second script renders the JSON as the classic 53-week grid of rounded boxes that reveal themselves diagonally.

Then a GitHub Actions workflow runs every morning, re-scrapes, re-renders, and commits the new SVG back to the repo. The commit message ends with [skip ci] so the bot does not trigger itself in a loop.

The whole system is two Python scripts and one cron workflow. No server, no database, no hosting bill. The repo is the database and GitHub Actions is the server.

Why not use a stats widget service?

Plenty of hosted README widgets exist. They also rate-limit, go down, and leave a broken-image icon on your profile at the worst moment. Generating your own SVGs means everything is committed to your repo, loads instantly, and looks exactly how you designed it.

All the code is public in my profile repository. Fork it, swap in your own username and photo, and your profile can wake up before you do.
